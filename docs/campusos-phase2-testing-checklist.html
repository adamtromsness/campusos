Phase 2 Testing Checklist

Walk through every workflow as each persona. For each item, note what works, what doesn't, and what feels wrong. Screenshots encouraged. This document becomes the input for the UI Design Guide.

How to test: Start the app (docker compose up -d, then npx nest start --watch in apps/api, then npx next dev in apps/web). Open http://localhost:3000. Log in as each persona using the email shown below. Test on desktop first, then try your phone (same URL if on the same network, or use browser DevTools mobile view).

1. Login & Home Page (All Personas)

Log in as each user below. The home page should be a clean Google-style launchpad with greeting, search bar, and app grid. Test each persona separately.
T-001 · Teacher Login
Log in as teacher@demo.campusos.dev
Expect: "Good morning/afternoon, James" greeting. App grid shows: Classes, Messages, Announcements. CampusOS logo in sans-serif. No clutter — just the launchpad.
T-002 · Parent Login
Log in as parent@demo.campusos.dev
Expect: "Good morning/afternoon, David" greeting. App grid shows: My Children, Messages, Announcements.
T-003 · Student Login
Log in as student@demo.campusos.dev
Expect: "Good morning/afternoon, Maya" greeting. App grid shows: My Classes, Messages, Announcements.
T-004 · Admin Login
Log in as admin@demo.campusos.dev
Expect: Greeting for the admin user. App grid shows all available apps. Admin should see everything.
T-005 · Home Page Design
Review the home page layout and feel across all personas
Is it clean and uncluttered? Does it feel like a launchpad? Is the CampusOS logo in sans-serif? Does the search bar look right? Are the app cards well-spaced?
T-006 · Sidebar Navigation
Check the sidebar for each persona
Sidebar should show the same apps as the home grid. Does it collapse on mobile? Do all links work (no 404s)? Are badge counts visible for Messages and Announcements?
T-007 · Notification Bell
Check the notification bell in the top bar for each persona
Bell icon should show a badge count if there are unread notifications. Click it — does the dropdown appear? Are notifications listed with correct types? Do the links go to the right place?

2. Teacher Workflows

Teacher — teacher@demo.campusos.dev

2A. Classes & Attendance

T-010 · View Classes
Click "Classes" from the home grid or sidebar
See a list/grid of the teacher's assigned classes (should be 6 classes). Each should show class name, section, period.
T-011 · Take Attendance
Click a class → go to the Attendance tab → mark a student tardy or absent → submit
See student roster with attendance status. Tap a student to change status (Present → Tardy → Absent). Add a note. Submit the attendance. Should see a success confirmation.
T-012 · Attendance Submitted State
After submitting, go back to the class and look at attendance again
The attendance should show as submitted/confirmed. Can you edit it after submitting? Should you be able to?

2B. Assignments

T-020 · View Assignments
Click a class → go to the Assignments tab
See a list of assignments for this class with title, type, due date, points, submission count.
T-021 · Create Assignment
Click "Create Assignment" → fill in title, type, due date, points → save
Form should have: title, type dropdown, category dropdown, instructions, due date picker, max points, extra credit toggle. Should save and appear in the list.
T-022 · Edit Assignment
Click an existing assignment → edit it → save
Can you change the title, due date, points? Does it save correctly?
T-023 · Category Weights
Open the category weight manager for a class
Should see categories (Homework, Assessments, Participation) with weights that sum to 100%. Can you adjust them? Does validation work?

2C. Grading

T-030 · Gradebook View
Click a class → go to the Gradebook tab
Spreadsheet-style grid: students as rows, assignments as columns. Cells should show grades with color coding. Right column shows current average.
T-031 · Enter a Grade
Click a cell in the gradebook → enter a grade → save
Inline grade entry. Does it save? Does the average update? Is the color coding correct?
T-032 · Publish Grades
Publish a grade (individual or batch "Publish All")
Published grades should become visible to students/parents. Unpublished grades should be hidden from them. Does the publish button work?
T-033 · Submission Review
Go to an assignment → click "Submissions" → review a student's submission
See a list of submissions with status. Click one to see the student's work. Enter grade + feedback. Save.
T-034 · Progress Notes
From the gradebook, open a student's progress note → write a note → save
Note should save. Toggle "Visible to parent" on. This should later show up on the parent's view.

2D. Teacher Messaging

T-040 · View Inbox
Click "Messages" from sidebar or home
See a list of message threads. Each shows participants, subject, preview, timestamp, unread badge.
T-041 · Read a Thread
Click a thread to open it
Messages displayed in chronological order. Own messages styled differently from others. Unread badge should clear after opening.
T-042 · Compose New Message
Click "Compose" → select thread type → pick recipient → write message → send
Thread type picker, recipient search, subject, body. Send should create the thread and navigate to it.
T-043 · Reply to a Message
Open an existing thread → type a reply → send
Reply appears in the thread. The other participant should see an unread badge when they log in.

2E. Teacher Announcements

T-050 · View Announcements
Click "Announcements" from sidebar or home
List of announcements. Each shows title, audience badge, timestamp, read/unread state.
T-051 · Create Announcement
Click "Create" → fill in title, body, audience type → publish
Audience picker (All School, Class, Year Group, Role). On publish, the announcement should appear in the feed. Stats should be visible to the author.

3. Parent Workflows

Parent — parent@demo.campusos.dev
T-060 · View Children
Click "My Children" from the home grid
See Maya Chen's card with grade level, attendance summary, and current grades.
T-061 · Child Attendance
Click Maya → view attendance
Calendar view showing attendance history. Green/amber/red for present/tardy/absent. Click a day for detail.
T-062 · Submit Absence Request
Click "Report Absence" → fill in date, reason → submit
Form with date picker, reason dropdown, text field. Should submit and show as pending.
T-063 · View Grades
Click Maya → view grades
Per-class averages. Click a class to see per-assignment breakdown with grades and feedback.
T-064 · Progress Notes
Check if teacher progress notes are visible on Maya's grade view
If the teacher wrote a parent-visible note, it should appear here with the teacher's name, effort rating, and text.
T-065 · Parent Messages
Go to Messages → read and reply to the teacher's thread
See the conversation with the teacher. Reply should go through content moderation. Badge should clear after reading.
T-066 · Parent Announcements
Go to Announcements
See school-wide and parent-targeted announcements. Should NOT see teacher/admin-only drafts. Mark as read should work.
T-067 · Parent Notifications
Check the notification bell and /notifications page
Should see notifications for: tardy marks, published grades, new messages, announcements. Each should link to the right place when clicked.

4. Student Workflows

Student — student@demo.campusos.dev
T-070 · View My Classes
Click "My Classes" from the home grid
See enrolled classes with current averages.
T-071 · View Assignments
Go to Assignments (sidebar or within a class)
See upcoming assignments across all classes. Overdue assignments should be flagged. Filter by class should work.
T-072 · Submit Assignment
Click an assignment → write response → submit
Text area for response. Submit button. Should show as "Submitted" after. If already graded, should show grade and feedback.
T-073 · View Grades
Go to Grades → click a class for detail
See per-class averages and per-assignment grades. Only published grades should be visible (no draft grades).
T-074 · Student Messages
Go to Messages
Should see threads the student participates in. Try composing — student should NOT be able to create a TEACHER_PARENT thread (wrong role).
T-075 · Student Announcements
Go to Announcements
Should see school-wide and student-targeted announcements. Should NOT be able to create announcements (no com-002:write).

5. Admin Workflows

Admin — admin@demo.campusos.dev
T-080 · Admin Home
Log in as admin → review the home page
Should see all apps. Admin should have the broadest access.
T-081 · Admin Classes View
Click Classes
Should see ALL classes in the school (not just assigned). Admin can view any class's attendance, assignments, gradebook.
T-082 · Admin Announcements
Create and publish a school-wide announcement
Should be able to create, publish, and view stats. Should see read count and delivery metrics.
T-083 · Admin Messages
Go to Messages → try viewing a thread you're not a participant in
Admin should see the thread in read-only mode with a "read-only" banner. This should be logged for FERPA audit.

6. Cross-Persona Flows

Multiple Personas
T-090 · Tardy Notification Flow
As teacher: mark Maya tardy → As parent: check notification bell
Parent should see a notification about Maya being marked tardy. Clicking it should go to the attendance detail. (May need to wait 30-60 seconds for Kafka + worker.)
T-091 · Grade Publish Notification
As teacher: grade and publish an assignment → As parent: check notifications → As student: check notifications
Both parent and student should receive a grade notification. Clicking should go to the gradebook.
T-092 · Message Exchange
As teacher: send message to parent → As parent: read and reply → As teacher: see the reply
Full round-trip messaging. Unread badges should appear and clear correctly on both sides.
T-093 · Content Moderation
As any user: try sending a message with a blocked keyword
Message should be rejected with a policy violation message. No detail about which keyword triggered it.
T-094 · Announcement Reach
As admin: create an ALL_SCHOOL announcement → Check as teacher, parent, student
All three personas should see the announcement. Mark-as-read should work for each. Admin stats should show read count increasing.

7. Permission Denials

Security Checks
T-100 · Student Cannot Grade
As student: try to access the gradebook grid or grade entry for any class
Should be denied — student can only see their own grades, not the full gradebook.
T-101 · Parent Cannot Submit Assignments
As parent: try to access the assignment submission form
Should be denied or not visible.
T-102 · Student Cannot Create Announcements
As student: go to Announcements → look for a "Create" button
Create button should not be visible. If you navigate to /announcements/new directly, should see a permission error.
T-103 · Parent Cannot See Other Students
As parent: try to access a student record that isn't Maya
Should be denied. Parent should only see their linked children.

8. Design & UX Review

T-110 · Overall Design Feel
Step back and evaluate the overall look and feel
Does it feel professional? Approachable? Clean? Would a teacher want to use this every day? Would a parent feel comfortable?
T-111 · Color Palette
Look at the colors throughout the app
Are the colors consistent? Does the blue/campus palette feel right? Are status colors (green/amber/red) clear?
T-112 · Typography
Look at the text throughout — headings, body, labels, buttons
Is it readable? Is the hierarchy clear (what's a heading vs body vs label)? Font sizes appropriate?
T-113 · Spacing & Density
Look at the whitespace, padding, margins throughout
Too cramped? Too spacious? Does each screen feel balanced? Are cards, tables, and lists well-spaced?
T-114 · Mobile Experience
Open the app on your phone or use browser DevTools mobile view
Does the sidebar collapse? Can you navigate? Can a parent check attendance at school pickup? Can a teacher take attendance on a tablet?
T-115 · Empty States
Look for any screens that show no data — empty inbox, no assignments, etc.
Empty states should have a friendly message and an icon, not a blank screen or an error.
T-116 · Loading States
Watch for loading indicators when pages load
Is there a spinner or skeleton? Or does the page just flash/jump? Does anything feel slow?
T-117 · Error Handling
Try things that should fail — wrong data, missing fields, etc.
Are error messages clear and helpful? Do they appear in the right place? No raw error codes or stack traces visible to the user?
