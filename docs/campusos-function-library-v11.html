CampusOS Function Library

148 functions across 28 groups — the complete access control catalogue
March 2026 · ERD v11 Aligned

About This Document

This document defines the CampusOS Function Library: 148 functions organised into 28 groups. Each function maps directly to an IAM access policy and represents a controllable unit of system access.

Purpose: define the complete set of functions that appear on the Request Access page. Users browse functions by group, see their current access, and request additional access. Approvers grant or deny with mandatory comments on denial.

Each function supports three access tiers: View Only (read access), Read/Write (operational access), and Admin (configuration access). Admin always includes Read/Write. Access grants are time-bound with optional expiry dates.

These functions will be mapped to the ERD v10 module structure and implemented as permission sets in the IAM subsystem. The function code (e.g. ATT-001) becomes the permission prefix in iam_role_assignment.

1. Function Library Overview

The CampusOS Function Library defines 148 functions organised into 28 groups. Each function maps directly to an IAM access policy in the system. Functions are the unit of access control — when a user requests access, they request a specific function at a specific tier.

Access Tiers

Every function supports three access tiers:

  • View Only — Read access. The user can see data but cannot create, modify, or delete anything. This is the minimum tier and is appropriate for oversight, reference, and transparency.
  • Read/Write — Operational access. The user can perform the function’s core operations: creating records, processing workflows, entering data, and completing tasks. This is the standard tier for someone doing the work.
  • Admin (Configure) — Configuration access. The user can change the function’s settings, policies, rules, and templates. This is the governance tier — it controls how the function behaves for everyone. Admin access always includes Read/Write capabilities.

How Access Requests Work

From the Request Access page, a user can:

  • Browse all functions organised by group.
  • See which functions they currently have access to, at what tier, and when the access expires.
  • See which functions they do not currently have access to.
  • Request access to any function at any tier. The request includes a justification.
  • The request routes through the appropriate approval chain (department head, building administrator, or district administrator depending on the function and tier).
  • If access is approved, it is granted with an effective date and optional expiry date.
  • If access is denied or revoked, the approver must provide a comment that is visible to the user.

Summary

148 functions across 28 groups. Each function has View Only, Read/Write, and Admin tiers. Every access grant is logged in the audit trail.

2. Attendance & Daily Operations

Tracking student presence, managing late arrivals, early dismissals, and daily procedures.

5 functions in this group:

ATT-001: Attendance Taking

Recording student attendance for classes, homeroom, and events.

  • View Only: View attendance records and history for students in your scope.
  • Read/Write: Mark students present, absent, tardy, or excused. Record attendance for assigned classes.
  • Admin: Configure attendance policies, define absence categories, set chronic absenteeism thresholds, and manage automated follow-up rules.

ATT-003: Early Dismissal

Processing students leaving before the end of the school day.

  • View Only: View early dismissal records.
  • Read/Write: Process early pickup requests, verify authorised adults against custody records, and release students.
  • Admin: Configure early dismissal policies and authorisation requirements.

ATT-004: Absence Follow-Up

Contacting families about unexplained absences.

  • View Only: View the follow-up queue and contact history.
  • Read/Write: Make contact attempts, log outcomes, and escalate unresolved absences.
  • Admin: Configure follow-up timelines, automated outreach rules, and escalation chains.

  • View Only: View active passes and pass history.

  • Admin: Configure pass limits, duration defaults, and destination options.
